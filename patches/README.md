# DDO Client Binary Mod Documentation

This document traces the modification history of the Dragon's Dogma Online (DDO) Windows client executable. The original packed client was dumped into an unpacked format using Ghidra, Scylla, etc. 
Modifications from 2020–2022 focused on removing anti-tampering, anti-cheating, and anti-debugging protections to enable research. 
Later modifications (2025 onwards) shifted toward quality-of-life improvements.

All binary diffs are recorded as `.r2` scripts (radiff2 from radare2) and accompanying `.json` dumps, which provide additional metadata. 
SHA-256 hashes are tracked at each stage to maintain a verifiable chain from the vanilla client through every successive patch.

## How to Apply Patches

Patches in this repository use the `.r2` format generated by [`radiff2 -r`](https://book.rada.re/tools/radiff2/intro.html), which produces a radare2 script of `wx` (write hex) commands at fixed offsets. 

To apply a patch, use `radare2` with the following flags:

```bash
radare2 -qnw -i <patch>.r2 <target>.exe
```

| Flag | Purpose |
|------|---------|
| `-q` | Quit after executing the script |
| `-n` | Do not load/parse binary headers |
| `-w` | Open the file in read-write mode |
| `-i` | Specify the `.r2` script to run |

To generate a new patch between two binaries:

```bash
radiff2 -r original.exe modified.exe > patch.r2
```

The accompanying `.json` files were generated with `radiff2 -j` ([data diffing docs](https://book.rada.re/tools/radiff2/datadiff.html)) and contain additional metadata such as file sizes, SHA-256 hashes, and the changed byte ranges.

Always verify the result after applying:

```bash
rahash2 -a sha256 patched.exe
```

### Caveats of layered patching

The `.r2` format is a simple sequence of "write byte X at offset Y" commands.

This means:
- **Order matters**
- **Applying a patch to the wrong base will silently corrupt the binary**

Each patch in the tables lists a Starting Hash and a Target Hash. The patch will only produce the expected Target Hash when applied to a binary that exactly matches the Starting Hash.
