DWARF debug symbol analysis and C++ definition generation for Dragon's Dogma Online reverse engineering using Ghidra.

== Overview

Analyzes DWARF debug symbols from Dragon's Dogma Online binary files. Extracts type information and generates C++ struct definitions through Ghidra's headless analysis API.

=== Functionality

* Symbol discovery in Ghidra data type manager
* C++ struct definition generation from binary data types
* C++ inheritance relationship processing via `super_` field analysis
* Template parameter extraction from templated types
* Namespace resolution from DWARF category paths
* Topological dependency sorting
* Headless execution through PyGhidra API

== Usage

=== Command Line Interface

The analyzer is invoked through the unified Ghidra analysis runner:

[source,bash]
----
./run_ghidra_analysis.sh dwarf SYMBOL_NAME PROGRAM_NAME [LOG_LEVEL] [EXPLORE_DEPTH]
----

**Parameters:**

* `SYMBOL_NAME` - DWARF symbol to analyze (e.g., "rLandInfo")
* `PROGRAM_NAME` - Program binary to analyze (e.g., "DDOORBIS.elf")
* `LOG_LEVEL` - Output verbosity: "DEBUG" or "INFO" (default: "INFO")
* `EXPLORE_DEPTH` - Maximum exploration depth 1-10 (default: 5)

**Examples:**

[source,bash]
----
# Basic symbol analysis
./run_ghidra_analysis.sh dwarf "rLandInfo" "DDOORBIS.elf"

# Debug logging with custom exploration depth
./run_ghidra_analysis.sh dwarf "MtObject" "DDOORBIS.elf" "DEBUG" 3

# With verbose Ghidra output
./run_ghidra_analysis.sh dwarf "rLandInfo" "DDOORBIS.elf" --verbose
----

=== Environment Configuration

The system uses environment variables for configuration:

[source,bash]
----
# Required
export SYMBOL_NAME="rLandInfo"
export LOG_LEVEL="INFO"                    # DEBUG | INFO
export EXPLORE_DEPTH=5                     # 1-10
----

=== Output Formats

**INFO Level (C++ Definitions Only):**

[source,cpp]
----
// Nested type definitions:
struct MtObject {
  // Packed structure (pack value: 0)
  _func_int** _vptr$MtObject;  // offset: 0, size: 8
};  // Total size: 8 bytes

struct cResource : public MtObject {
  // Packed structure (pack value: 0)
  uint32_t mMagicID : 16;  // offset: 8, size: 2
  char mPath[64];          // offset: 12, size: 64
  int32_t mRefCount;       // offset: 76, size: 4
};  // Total size: 112 bytes

// Main type definition:
struct rLandInfo : public cResource {
  MtTypedArray<rLandInfo::cLandInfo> mLandInfo;  // offset: 112, size: 32
};
----

**DEBUG Level Output:**
- Symbol discovery steps
- Type exploration details
- Namespace extraction data
- Inheritance relationship data
- Template parameter processing
- Execution metrics
