Function analysis tool using PyGhidra for Dragon's Dogma Online binary reverse engineering.

== Overview

Decompiles C++ functions from Dragon's Dogma Online binaries. Implements recursive decompilation, namespace parsing, cross-reference analysis, and struct dependency mapping via Ghidra's headless API.

=== Components

* Recursive function call analysis with configurable depth limits
* C++ namespace parsing and relationship mapping
* Cross-reference analysis using Ghidra's ReferenceManager API
* DWARF-aware struct dependency detection
* Service-oriented architecture with dependency injection
* PyGhidra headless execution via command line interface
* Two output modes: INFO (code only) and DEBUG (analysis metadata)

----

== Usage

=== Command Line Interface

The analyzer is invoked through the unified Ghidra analysis runner:

[source,bash]
----
./run_ghidra_analysis.sh function FUNCTION_NAME PROGRAM_NAME [LOG_LEVEL] [MAX_DEPTH]
----

**Parameters:**

* `FUNCTION_NAME` - Function to analyze (e.g., "rLandInfo::load")
* `PROGRAM_NAME` - Program binary to analyze (e.g., "DDOORBIS.elf")
* `LOG_LEVEL` - Output verbosity: "DEBUG" or "INFO" (default: "INFO")
* `MAX_DEPTH` - Maximum recursion depth 1-10 (default: 3)

**Examples:**

[source,bash]
----
# Default analysis
./run_ghidra_analysis.sh function "rLandInfo::load" "DDOORBIS.elf"

# Debug output with increased depth
./run_ghidra_analysis.sh function "rLandInfo::load" "DDOORBIS.elf" "DEBUG" 5

# Single-level analysis
./run_ghidra_analysis.sh function "MtObject::create" "*" "INFO" 1
----

=== Environment Configuration

The system uses environment variables for configuration:

[source,bash]
----
# Required
export FUNCTION_NAME="rLandInfo::load"
export LOG_LEVEL="INFO"                    # DEBUG | INFO
export MAX_DEPTH=3                         # 1-10
----

== API Reference

=== Core Classes

==== FunctionDecompilationAnalyzer

Main facade class for function analysis.

[source,python]
----
from function_analyzer.analysis import FunctionDecompilationAnalyzer

analyzer = FunctionDecompilationAnalyzer(program, log_level="INFO")
analyzer.analyze_function("rLandInfo::load", max_depth=3)
----

==== FunctionSearchService  

Discovers and matches functions with namespace support.

[source,python]
----
from function_analyzer.services import FunctionSearchService

search_service = FunctionSearchService(program, logger)
function = search_service.find_function_by_name("rLandInfo::load")
qualified_name = search_service.get_qualified_function_name(function)
----

==== DecompilationService

Extracts and processes decompiled C code.

[source,python]
----
from function_analyzer.services import DecompilationService

decompiler = DecompilationService(program, logger)
code = decompiler.get_decompiled_function(function)
calls = decompiler.get_function_calls_from_decompiled_code(code)
----

==== CrossReferenceAnalyzer

Finds function relationships and struct dependencies.

[source,python]
----
from function_analyzer.services import CrossReferenceAnalyzer

analyzer = CrossReferenceAnalyzer(program, logger)
related = analyzer.analyze_related_functions(function, namespace_filter="rLandInfo")
----

==== StructAnalysisService

DWARF and struct dependency analysis.

[source,python]
----
from function_analyzer.services import StructAnalysisService

struct_analyzer = StructAnalysisService(program, logger)
dwarf_info = struct_analyzer.analyze_function_with_dwarf(function)
related_types = struct_analyzer.get_related_datatypes("rLandInfo")
----

=== Configuration

==== AnalyzerConfig

----

# Recursion settings
AnalyzerConfig.DEFAULT_RECURSION_DEPTH  # 3
AnalyzerConfig.MAX_RECURSION_DEPTH      # 10

# Decompilation settings
AnalyzerConfig.DECOMPILATION_TIMEOUT    # 30 seconds

# Function scoring weights
AnalyzerConfig.SCORE_NAMESPACE_BONUS    # 10
----

=== Utilities

==== NamespaceUtils

C++ namespace parsing utilities.

[source,python]
----
from function_analyzer.utils import NamespaceUtils

# Extract namespace from function signature
namespace = NamespaceUtils.extract_namespace_from_signature(signature)

# Extract function calls from decompiled C code
calls = NamespaceUtils.extract_function_calls(decompiled_code)

# Check namespace relationship
related = NamespaceUtils.is_namespace_related(text, namespace)
----
